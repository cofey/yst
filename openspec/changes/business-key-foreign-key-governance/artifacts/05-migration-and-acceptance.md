# 迁移与验收

## 1. 迁移前数据巡检清单

1. 主表业务主键空值检查（`xxx_id IS NULL OR xxx_id = ''`）。
2. 主表业务主键重复检查（`GROUP BY xxx_id HAVING COUNT(*) > 1`）。
3. 关联表孤儿数据检查（关联列在主表不存在）。
4. 业务主键格式检查（UUID 格式合法性）。

## 2. 分阶段发布方案

### Phase A: Schema
- 新建/调整主表主键为 `xxx_id CHAR(36)`。
- 新建/调整关联表外键为业务键并建立外键约束。
- 执行巡检并清理异常数据。

### Phase B: Contract
- 后端接口改为业务键入参/出参。
- 前端类型、路由、操作参数切换业务键。
- 联调验证所有 CRUD 与授权流程。

### Phase C: Governance
- CI 增加 DTO/VO 字段门禁（禁止数值型主键暴露）。
- SQL 评审门禁（主键/外键必须为业务键）。
- 发布后观察日志与错误告警，完成收敛。

## 3. 回归验收矩阵

| 领域 | 场景 | 预期 |
|---|---|---|
| 用户 | 新增用户 | 生成并持久化 `user_id`，可查询可编辑 |
| 用户 | 删除用户 | 有关联时阻断；无关联时删除成功 |
| 角色 | 角色授权菜单 | `sys_role_menu` 使用 `role_id/menu_id` 正常保存 |
| 菜单 | 菜单增删改查 | `menu_id` 贯穿全链路 |
| 公司 | 用户绑定公司 | `sys_user_company` 使用 `user_id/company_id` 正常保存 |
| 鉴权 | 登录与鉴权 | 登录态和权限查询以 `user_id` 为标识 |
| 前端 | 列表与详情跳转 | 参数全部为业务键字符串 |
| 接口 | 契约检查 | 对外响应不出现内部数值型主键 |
