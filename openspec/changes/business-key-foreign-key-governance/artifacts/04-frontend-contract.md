# 前端契约改造规范

## 1. 实体类型规范

1. 所有实体主标识类型统一为 `string`。
2. 字段命名与后端保持一致：`userId`、`roleId`、`menuId`、`companyId`。
3. 前端禁止使用 `id: number` 作为业务标识。

## 2. 路由与交互规范

1. 路由参数使用业务键（例如 `/users/:userId/edit`）。
2. 表格 `row-key` 使用业务键字段。
3. 删除、编辑、详情等操作参数统一使用业务键。

## 3. 接口适配层规则

1. API 请求参数不传内部数值型主键。
2. API 响应若出现内部数值型主键，适配层必须过滤且记录告警。
3. 前端状态管理以业务键为唯一索引键。
