## ADDED Requirements

### Requirement: 系统必须按当前用户权限返回菜单树

系统 MUST 基于当前登录用户关联角色的菜单授权，返回可用于导航渲染的菜单树数据。

#### Scenario: 普通用户菜单可见范围
- **WHEN** 普通用户请求当前用户菜单树
- **THEN** 仅返回其角色已授权的可见菜单
- **AND** 不返回无权限菜单

#### Scenario: 管理员菜单可见范围
- **WHEN** 管理员请求当前用户菜单树
- **THEN** 返回全部可见菜单（满足状态与可见性条件）

### Requirement: 前端顶栏必须由权限菜单树全量动态渲染

前端顶栏菜单 MUST 不再写死，必须由后端返回的权限菜单树驱动渲染。

#### Scenario: 全量动态渲染
- **WHEN** 用户登录并获取权限菜单树
- **THEN** 顶栏菜单全部从菜单树渲染
- **AND** 菜单层级关系与顺序与返回树保持一致

#### Scenario: 前端路由缺失项处理
- **WHEN** 菜单树存在前端尚未注册的路由 path
- **THEN** 前端仍应渲染该菜单项（不做预过滤）

### Requirement: 无权限路由访问仅提示不强制拦截

#### Scenario: 手动访问无权限路由
- **WHEN** 用户手动输入无菜单权限的页面地址
- **THEN** 系统提示无权限
- **AND** 不进行强制跳转拦截
