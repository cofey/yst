# 代码质量使用规范（本地检查 + 提交前门禁）

适用范围：`yst-server/`（Java）与 `yst-ui/`（Vue3 + TS）。

## 1. 工具范围

后端（Java）：

- `Checkstyle`：代码风格与基础规范。
- `SpotBugs`：静态缺陷检查。
- `SonarQube`：本地触发代码质量扫描（按需）。

前端（Vue3 + TS）：

- `ESLint`：代码规则检查。
- `Prettier`：代码格式检查。

## 2. 本地检查命令

## 2.1 后端（在 `yst-server/` 执行）

完整检查：

```bash
mvn clean verify
```

单项检查：

```bash
mvn checkstyle:check
mvn spotbugs:check
```

SonarQube 扫描（按需执行）：

```bash
mvn clean verify sonar:sonar \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.login=sqa_db2ab5271dda7a95252235b495d46e3aff314c56 \
  -Dsonar.projectKey=yst-server-ci
```

## 2.2 前端（在 `yst-ui/` 执行）

完整检查：

```bash
npm run check
```

等价命令：

```bash
npm run lint
npm run format
```

自动修复：

```bash
npm run lint:fix
npm run format:write
```

说明：`lint` 已配置 `--max-warnings=0`，出现 warning 也视为失败。

## 3. 提交前门禁（强制）

按改动范围执行以下门禁：

1. 仅改后端：`cd yst-server && mvn clean verify`
2. 仅改前端：`cd yst-ui && npm run check`
3. 同时改前后端：两条命令都必须通过

门禁规则：

- 任一命令失败，不允许提交。
- 禁止带着 Checkstyle/SpotBugs/ESLint/Prettier 问题提交。

## 4. 推荐提交前顺序

1. 先修复格式：

```bash
cd yst-ui && npm run format:write && npm run lint:fix
```

2. 再执行严格检查：

```bash
cd yst-server && mvn clean verify
cd yst-ui && npm run check
```

3. 全通过后再 `git commit`

## 5. 快速命令清单

```bash
# 后端
cd yst-server
mvn clean verify

# 前端
cd yst-ui
npm run check

# 前端自动修复
npm run lint:fix
npm run format:write
```
